# WebSocket 协议速查表

## 📋 帧格式快速参考

### 基本结构
```
[FIN][RSV1-3][Opcode] [MASK][Payload Len] [Masking Key] [Payload Data]
 1bit   3bit    4bit    1bit    7bit        0/4 bytes     0-2^63 bytes
```

### 操作码对照表
| Opcode | 类型 | 说明 |
|--------|------|------|
| 0x0 | CONTINUATION | 连续帧 |
| 0x1 | TEXT | 文本帧 |
| 0x2 | BINARY | 二进制帧 |
| 0x8 | CLOSE | 关闭连接 |
| 0x9 | PING | 心跳请求 |
| 0xA | PONG | 心跳响应 |

## 🔢 长度编码规则

```
长度值范围    | 编码方式          | 字节数
0-125        | 直接编码          | 1字节
126-65535    | 后续2字节16位长度  | 3字节
65536-2^63   | 后续8字节64位长度  | 9字节
```

## 🎯 常用报文格式

### 文本消息 "Hello"
**发送（服务端→客户端）：**
```
81 05 48 65 6c 6c 6f
│  │  └───── "Hello"
│  └──── 长度=5
└──── FIN=1, TEXT
```

**接收（客户端→服务端）：**
```
81 85 37 fa 21 3d 7f 9f 4d 51 58
│  │  └─── 掩码键    └─── 掩码数据
│  └──── MASK=1, 长度=5
└──── FIN=1, TEXT
```

### 分片消息
```
开始帧: 01 06 48 65 6c 6c 6f 20     (Hello )
连续帧: 00 05 57 6f 72 6c 64        (World)
结束帧: 80 01 21                    (!)
```

### 控制帧
```
Ping: 89 05 48 65 6c 6c 6f          (Hello)
Pong: 8a 05 48 65 6c 6c 6f          (Hello)  
Close: 88 02 03 e8                  (状态码1000)
```

## ⚡ 核心算法

### 掩码解密
```java
data[i] ^= mask[i % 4]
```

### 长度解析
```java
// 16位长度
length = (buffer[0] << 8) | buffer[1]

// 64位长度  
length = 0
for(i=0; i<8; i++)
    length = (length << 8) | buffer[i]
```

### 帧头构建
```java
// 第一字节
header[0] = (fin ? 0x80 : 0) | (opcode & 0x0F)

// 第二字节  
header[1] = (mask ? 0x80 : 0) | (length <= 125 ? length : length <= 65535 ? 126 : 127)
```

## ⚠️ 常见陷阱

1. **字节序**：网络传输使用大端序
2. **掩码要求**：客户端→服务端必须掩码
3. **控制帧限制**：长度必须<125字节
4. **分片规则**：只有第一帧设置opcode
5. **状态码**：关闭帧可携带状态码(2字节)
