# RFC 7230-7235: HTTP/1.1 标准 (2014)

**状态**: HTTP/1.1 最新标准
**替代**: RFC 2616

## 概述

RFC 7230-7235 是 HTTP/1.1 的现代化规范，将 RFC 2616 拆分为6个独立文档：

| RFC | 标题 | 描述 |
|-----|------|------|
| RFC 7230 | Message Syntax and Routing | 消息语法和路由 |
| RFC 7231 | Semantics and Content | 语义和内容 |
| RFC 7232 | Conditional Requests | 条件请求 |
| RFC 7233 | Range Requests | 范围请求 |
| RFC 7234 | Caching | 缓存 |
| RFC 7235 | Authentication | 认证 |

---

## RFC 7230: Message Syntax and Routing

### 核心变更 (vs RFC 2616)
- 引入 Augmented BNF (ABNF) 列表扩展
- 规范化 HTTP/1.1 消息格式
- 废除了 Transfer-Encoding: chunked 的简化语法

### 消息格式
```
HTTP-message = start-line
               *( header-field CRLF )
               CRLF
               [ message-body ]
```

### 请求行
```
Request-Line = method SP request-target SP HTTP-version CRLF
```

### 响应行
```
Status-Line = HTTP-version SP status-code SP reason-phrase CRLF
```

### 关键概念
- **host**: 唯一必需的头字段
- **connection**: 控制连接管理
- **transfer-encoding**: 传输编码定义

## RFC 7231: Semantics and Content

### 请求方法
| Method | Safe | Idempotent | Cacheable |
|--------|------|------------|-----------|
| GET | Yes | Yes | Yes |
| HEAD | Yes | Yes | Yes |
| POST | No | No | Yes (受限) |
| PUT | No | Yes | No |
| DELETE | No | Yes | No |
| CONNECT | No | No | No |
| OPTIONS | Yes | Yes | No |
| TRACE | Yes | Yes | No |

### 状态码
| 类别 | 范围 | 描述 |
|------|------|------|
| 1xx | 100-199 | Informational |
| 2xx | 200-299 | Successful |
| 3xx | 300-399 | Redirection |
| 4xx | 400-499 | Client Error |
| 5xx | 500-599 | Server Error |

### Content Negotiation
- **Accept**: 接受的媒体类型
- **Accept-Charset**: 接受的字符集
- **Accept-Encoding**: 接受的内容编码 (gzip, deflate, br)
- **Accept-Language**: 接受的语言

### 表示元数据
- **Content-Type**: 媒体类型
- **Content-Encoding**: 内容编码 (压缩)
- **Content-Language**: 内容语言
- **Content-Location**: 内容位置

---

## RFC 7232: Conditional Requests

### 验证器 (Validators)
两种形式：
1. **Last-Modified**: 时间戳
2. **ETag**: 实体标签

### 弱/强验证器
- **强验证器**: 必须唯一标识每个表示版本
- **弱验证器**: 允许相同内容不同表示共享

### 条件头字段
| Header | 用途 |
|--------|------|
| If-Match | 防止"丢失更新"问题 |
| If-None-Match | 缓存验证 |
| If-Modified-Since | 条件获取 |
| If-Unmodified-Since | 乐观锁 |
| If-Range | 范围请求条件 |

### 优先级
1. If-Match (最强)
2. If-Unmodified-Since
3. If-None-Match
4. If-Modified-Since

---

## RFC 7233: Range Requests

### 范围请求
允许请求资源的部分内容：

```
Range: bytes=0-499
Range: bytes=-500 (最后500字节)
Range: bytes=500-
```

### 响应状态码
- **206 Partial Content**: 成功返回范围
- **416 Range Not Satisfiable**: 范围无效

### Content-Range
```
Content-Range: bytes 21010-47021/47022
```

### 多部分字节范围
```
Content-Type: multipart/byteranges; boundary=分割字符串
```

## RFC 7234: Caching

### 缓存指令 (Cache-Control)

**请求指令**:
| Directive | 描述 |
|-----------|------|
| max-age | 接受的最大年龄 |
| min-fresh | 最小新鲜时间 |
| no-cache | 必须重新验证 |
| no-store | 不存储 |
| no-transform | 不转换 |
| only-if-cached | 仅从缓存 |

**响应指令**:
| Directive | 描述 |
|-----------|------|
| public | 可存储在任何缓存 |
| private | 仅私有缓存 |
| max-age | 保鲜时间 |
| s-maxage | 共享缓存最大年龄 |
| must-revalidate | 必须重新验证 |
| proxy-revalidate | 代理必须重新验证 |
| no-store | 不存储 |
| no-transform | 不转换 |

### 新鲜度计算
```
freshness_lifetime > current_age → 保鲜
```

### 验证流程
1. 发送条件请求 (If-None-Match / If-Modified-Since)
2. 200 OK: 使用新内容
3. 304 Not Modified: 使用缓存

## RFC 7235: Authentication

### 认证框架
- **Challenge**: 服务器发送的挑战
- **Credentials**: 客户端凭证

### 状态码
| Code | 描述 |
|------|------|
| 401 Unauthorized | 需要认证 |
| 407 Proxy Authentication Required | 代理需要认证 |

### 认证头字段
| Header | 方向 | 描述 |
|--------|------|------|
| WWW-Authenticate | Server → Client | 认证挑战 |
| Authorization | Client → Server | 客户端凭证 |
| Proxy-Authenticate | Proxy → Client | 代理认证挑战 |
| Proxy-Authorization | Client → Proxy | 代理凭证 |

### 认证方案
- Basic: 用户名:密码 Base64编码
- Digest: MD5哈希
- Bearer: Token认证

---

## 关键变化 (对比 RFC 2616)

### RFC 7230
- 引入 BNF 列表扩展 (#rule)
- 规范化 host 头字段
- 移除 Transfer-Encoding 简写

### RFC 7231
- OPTIONS 方法更明确
- 移除 305 Use Proxy
- 引入 426 Upgrade Required

### RFC 7232
- 规范化 ETag 格式 (移除引号必需)
- 定义验证器比较优先级
- 允许弱 ETag 用于缓存

### RFC 7233
- 允许单一范围的 multipart/byteranges
- 移除 range-unit 限制

### RFC 7234
- 移除 Expires 一年限制
- 允许启发式新鲜度用于查询 URI
- 规范化 Age 计算

### RFC 7235
- 引入 Authentication Scheme Registry
- 允许 challenge 不带 realm
