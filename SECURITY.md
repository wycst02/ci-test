# 安全策略

## 报告漏洞

wastnet 重视安全性。如果你发现了安全漏洞，建议优先通过邮件私下报告，以便我们在修复完成后再公开披露。如果问题不涉及安全风险，也可以直接提交 Issue。

- 发送邮件至：[shallxiao@126.com](mailto:shallxiao@126.com)

我们会在 **48 小时内**确认收到报告，并尽快跟进修复。

## 报告流程

1. 提交漏洞详情（包括复现步骤、影响范围、建议修复方案）
2. 我们确认并评估漏洞等级
3. 修复开发中...
4. 发布修复版本并公开致谢（如你同意）

## 受支持的版本

| 版本 | 支持状态 |
| --- | --- |
| 最新 SNAPSHOT | ✅ 积极开发中 |
| 正式版 | 待首次发布后补充 |

## 安全最佳实践

- 生产环境建议启用 SSL/TLS 加密传输
- 配置连接过滤器（`ConnectionFilter`）限制来源 IP
- 根据实际场景调整 HTTP 请求大小限制（`HttpConf`）
- 不要在暴露给外网的服务上开启 Debug 日志级别
